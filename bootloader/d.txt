АРХИТЕКТУРА СИСТЕМЫ
1. BOOT SEQUENCE (Limine → HAL → Kernel)
Этап 1: Limine Bootloader

Загружает kernel.elf в память
Переключает CPU в 64-bit long mode
Настраивает базовые page tables с HHDM (Higher Half Direct Map)
Инициализирует framebuffer (1024x768x32)
Передает управление на boot.asm
Этап 2: HAL Boot (boot.asm → boot_main.c)

boot.asm: Entry point, настройка стека, вызов boot_main()
boot_main.c:
Инициализация serial port (COM1) для логов
Получение Limine responses (framebuffer, terminal, HHDM offset)
Парсинг cmdline для определения режима (GUI/Terminal)
Вызов kernel_main() с параметрами
Этап 3: Kernel Initialization (kernel/src/lib.rs)

kernel_main() {
    1. HAL init (GDT, IDT)
    2. Memory init (PMM, VMM, Heap)
    3. IF GUI mode:
       - Scheduler init
       - IPC init
       - VFS init
       - Initrd load
       - Drivers init (keyboard, mouse)
       - Window manager init
    4. IF Terminal mode:
       - Keyboard driver only
    5. Enable interrupts
    6. Start UI loop / Terminal loop
}
2. HARDWARE ABSTRACTION LAYER (HAL)
Компоненты:

GDT (Global Descriptor Table): 5 сегментов (null, code, data, user code, user data)
IDT (Interrupt Descriptor Table): 256 entries для exceptions и IRQs
Interrupt Handlers: Timer (IRQ0), Keyboard (IRQ1), Mouse (IRQ12)
Context Switch: Сохранение/восстановление регистров процессов
Syscall Entry: Fast system call interface (syscall/sysret)
I/O Ports: Обертки для in/out инструкций
Интерфейс (hal.h):

void hal_init(void);
void hal_enable_interrupts(void);
void hal_disable_interrupts(void);
uint8_t hal_inb(uint16_t port);
void hal_outb(uint16_t port, uint8_t value);
3. MEMORY MANAGEMENT
Physical Memory Manager (PMM)

Bitmap allocator для физических страниц (4KB)
131072 страниц доступно (512MB RAM)
Функции: pmm_alloc_page(), pmm_free_page()
Virtual Memory Manager (VMM)

4-level page tables (PML4 → PDP → PD → PT)
Kernel mapped at 0xFFFFFFFF80000000 (higher half)
HHDM offset: 0xFFFF800000000000 (physical memory mapping)
Функции: map_page(), unmap_page(), translate()
ПРОБЛЕМА: VGA buffer (0xB8000) не замаплен через HHDM
Heap Allocator

16MB kernel heap
Linked list allocator
Поддержка alloc::vec::Vec, alloc::string::String
4. PROCESS MANAGEMENT
Process Structure:

struct Process {
    pid: usize,
    state: ProcessState,  // Ready, Running, Blocked, Terminated
    context: Context,     // Registers (rax, rbx, rip, rsp, etc.)
    page_table: *mut PageTable,
    priority: u8,
}
Scheduler:

Round-robin алгоритм
Ready queue для процессов
Context switching через context_switch.asm
Timer interrupt (IRQ0) для preemption

Message Passing:

struct Message {

    receiver: usize,

Функции: ipc_send(), ipc_receive(), ipc_reply()
Shared Memory:

struct SharedMemory {
    id: usize,

    size: usize,
    permissions: Permissions,
}
Функции: shm_create(), shm_attach(), shm_detach()

Структура:

/                    (root)
├── dev/             (devfs - устройства)

VFS Interface:

trait FileSystem {
    fn open(&self, path: &str) -> Result<FileHandle>;
    fn read(&self, handle: FileHandle, buf: &mut [u8]) -> Result<usize>;
    fn write(&self, handle: FileHandle, buf: &[u8]) -> Result<usize>;
    fn close(&self, handle: FileHandle) -> Result<()>;
}
Реализации:

MemFS: In-memory filesystem
DevFS: Device files (/dev/null, /dev/zero)
ProcFS: Process information (/proc/[pid])

9. TERMINAL MODE
Статус: НЕ РАБОТАЕТ (проблема с VGA)

Проблема:

VGA text mode buffer (0xB8000) недоступен
HHDM mapping (0xFFFF8000000B8000) вызывает page fault
Identity mapping (0xB8000) тоже не работает
Limine Terminal API недоступен в terminal mode
Текущая реализация:

Serial-only output (COM1)
Keyboard polling работает
Команды: help, ls, pwd, clear, exit
Prompt: root@dunit:~#
Решения:

Serial-only terminal (работает через COM порт)
Написать VGA driver с page table mapping
Убрать terminal mode, оставить только GUI

15. ТЕКУЩИЕ ПРОБЛЕМЫ
КРИТИЧЕСКИЕ:

Terminal Mode VGA: Не работает запись в VGA buffer
Причина: Limine не мапит 0xB8000 через HHDM
Решение: Написать VGA driver с page table setup
СРЕДНИЕ: 2. VMM не полностью реализован (пустой init) 3. Scheduler не запускается (нет процессов) 4. ELF loader не тестирован 5. Userspace программы не реализованы

МЕЛКИЕ: 6. Много unused imports warnings 7. Unsafe блоки в terminal loop 8. Hardcoded константы (VGA_WIDTH, colors)

Совместимость:

QEMU: работает

Real hardware: GUI mode должен работать, Terminal mode нет
ЗАКЛЮЧЕНИЕ
Dunit OS - это функциональный microkernel с GUI, работающим в QEMU. Основная проблема - Terminal Mode не работает из-за VGA mapping. Система имеет все базовые компоненты ОС: memory management, process scheduling, IPC, VFS, drivers. Следующий шаг - исправить Terminal Mode и запустить реальные userspace процессы.
